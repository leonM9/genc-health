slide 04

Privacy-by-Design Pillars (RA 10173 §20)

Four design pillars align with the four privacy obligations of the Philippine Data Privacy Act.
🔐 Encryption
AES-256-GCM at rest, performed client-side in the browser. Server never sees plaintext.
🛡️ Access Control
CP-ABE attribute policy: (Role:Doctor AND Dept:X) OR Owner:patient
🌐 Decentralization
IPFS storage + Merkle anchoring — no single point of trust or failure.
📋 Auditability
Every read/write cryptographically signed (ECDSA secp256k1).

slide 24

Scope Limitations & Boundaries

Simulated CP-ABE
Shunting-yard policy evaluator. Real lib (OpenABE) requires PBC pairings — out of thesis scope.
Simulated Hyperledger Besu
Merkle root + tx hash + block number generated locally. Same data structures as a real Besu chain.
Centralized IPFS Gateway
Uses Pinata. True decentralization needs self-hosted IPFS node (Kubo).
Web-Only (No Mobile App)
Future React Native + WalletConnect implementation recommended.
